
👉 Middleware is a filter that runs before or after a request reaches the controller.

👉 Simple meaning:

Middleware = Security/Checking layer between request and controller

🔄 Flow (Very Important)

User → Middleware → Controller → Response

🧠 Easy Real-Life Example

👉 Think of middleware like a security guard 🚔

User tries to enter (request)
Guard checks:
Logged in or not?
Has permission or not?
Then allows or blocks
📘 Why Middleware is Used?
✅ Authentication (login check)
✅ Authorization (permission check)
✅ Logging requests
✅ Prevent unauthorized access
✅ Security (CSRF protection)
📘 Example in Laravel
Route::get('/dashboard', function () {
    return "Welcome";
})->middleware('auth');

👉 Meaning:

Only logged-in users can access /dashboard
📘 Creating Middleware
php artisan make:middleware CheckAge
Example Middleware Code
public function handle($request, Closure $next)
{
    if ($request->age < 18) {
        return "Not Allowed";
    }

    return $next($request);
}

👉 If condition true → block
👉 Else → go to controller

📘 Types of Middleware
🔹 1. Global Middleware

👉 Runs on every request

🔹 2. Route Middleware

👉 Runs on specific routes

🔹 3. Group Middleware

👉 Runs on a group of routes
